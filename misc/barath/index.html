1. Requirements

  1. Build the given 3 screens from the UI mockup above using Flutter in 7 days.
  2. Focus on UI replication only - Replicate the visual design. Functionality is not required.
  3. Break down the screen into components - Build one component/screen per day. You decide the order, start with the easiest.
  4. Use AI freely - ChatGPT, Gemini, Claude, GitHub Copilot etc. are allowed and encouraged. No limitation in this.
  5. Match the design exactly - Colors, spacing, layout must be accurate.
  6. Use graphics from the source - For graphics/ background images, take screenshots from the given image and use it.
  7. Identify and use the correct font - Find the font used in the screen and apply it in your Flutter app.
  8. Use matching icons - Try to use icons that resemble as close to the screenshot as possible.
  9. Commit daily - Push your progress to GitHub every day.
  10. Don't skip days - Minimum one component per day.

2. Deliverables (Day 0 to Day 7)

Setup (Day 0)

  1. Install the latest stable version of Flutter: https://docs.flutter.dev/install/archive
  2. Create a public GitHub repository and email the link to yuva.dev.in@gmail.com

Day Plan (Day 1 to 7)

  1. Create 8 separate Flutter projects (day0/ through day7/)
  2. Each project must be a complete and runnable Flutter application
  3. Include a README.md to mention the changes that you have done for the day and your learnings
  4. Include screenshot.png in each folder
  5. Commit to GitHub daily

Note: Use the same GitHub repository link that you sent on Day 0. Do not create new repositories for each day.

3. Coding Guidelines

  1. All files and folder names throughout the assignment repo should be in snake_case
  2. Class names should be in PascalCase
  3. Class instances, functions, variables, field values should be in camelCase
  4. Code should be separated into simpler functions wherever possible. Deep nesting of code is not allowed.

4. Evaluation Criteria

You will be evaluated based on:

  1. Closest reproduction of the UI - How accurately your UI matches the mockup
  2. Ability to understand the code and modify it - Can you explain your code and make changes
  3. Ability to run without errors - All projects must run successfully
  4. Learning and understanding of Flutter - You should learn and have a basic understanding of Flutter and its components while working on this project
